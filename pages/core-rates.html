Core Rates

By BigStupidJellyfish | Last updated 2021-November-17

Cores are nice to have, so maps that give lots of cores must be nice. It takes something like 2000-3000 cores to craft a 5★ Para, so there's always a use for them - PR could happen this decade, and maybe even Singularity+ if we're lucky.

8-1N obviously gives no cores, but may be nice if you only want XP and want it fast. Heliport-spawned enemies generally don't drop dolls so 8-2 (SF dragging) doesn't give anything significant.

Cores per run isn't the only metric you should use. XP is also valuable, and each run will have different costs/requirements/time per run.

Autobattles#

Compared to corpse dragging, autobattles are horrendously resource-inefficient. If your time is limited and you've already got a 5-star parachute or two, they can be nice as a lazy way to level up the remainder of your armory. For general information including XP and resource efficiency, this excellent spreadsheet should be all you need. For core data, here are my observations:

Map Cores/run Cores/kWRSC Cores/hour Sample size
7-1 0.49 0.45 0.45 111 runs
11-5 1.28 0.42 0.53 107 runs
Wolf & Owl II EX (Isomer) 0.13 0.04 0.06 48 runs

where kWRSC is 1000 weighted resources (parts triple value). Run counts are currently low, so these numbers are somewhat rough. I'll update this over time as my sample size improves.
